# Kemou 博客内容发布指南

这个博客的页面会根据文章自动生成。以后新增文章或随笔时，不需要重新设计或重写首页、分类页、归档页和 RSS。

## 最简单的方式

你可以直接把文章正文、标题、配图和想放入的分类发给 Codex 或其他编程 Agent，让它按下面的格式放进网站并发布。

## 自己添加一篇文章

1. 在项目目录运行：

   ```powershell
   pnpm new-post -- my-new-post
   ```

   `my-new-post` 是文章的网址名称，建议只用小写英文、数字和连字符。

2. 打开自动生成的 `src/content/posts/my-new-post.md`，填写文章信息和正文：

   ```markdown
   ---
   title: 文章标题
   published: 2026-07-18
   description: 一两句话概括这篇文章
   image: /assets/images/my-cover.webp
   tags: [AI, 学习]
   category: 学习与项目
   draft: false
   ---

   从这里开始写正文。
   ```

3. 如果文章有图片，把图片放到 `public/assets/images/`，然后在文章里使用：

   ```markdown
   ![图片说明](/assets/images/my-image.webp)
   ```

4. 本地预览时运行 `pnpm dev`。确认无误后把改动推送到 GitHub，GitHub Pages 会自动更新网站。

## 文章与随笔怎么区分

技术上不需要做两套页面。随笔也是一篇 Markdown，只要把 `category` 写成 `生活随记`，或者以后新建一个分类名称即可。新分类、标签、文章数量、归档和 RSS 都会自动出现，不需要改前端代码。

## 常用字段

- `title`：文章标题。
- `published`：发布日期，格式为 `年-月-日`。
- `description`：首页卡片和搜索结果里显示的简介。
- `image`：封面图；不需要封面时可以留空。
- `tags`：可以写多个标签。
- `category`：分类名称；填写新名称就会自动新增分类。
- `draft`：设为 `true` 时暂不公开，准备发布时改为 `false`。

## 哪些修改仍然需要改网页

普通发文、添加图片、新增分类和标签都不需要改网页。只有新增一种全新的功能、改变页面布局、修改动画或整体美术风格时，才需要调整前端代码。
