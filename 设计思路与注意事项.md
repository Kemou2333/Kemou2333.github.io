# Kemou 博客拟态改造:设计思路与注意事项

> 这份文档概括了 2026-07-18 两轮 AI 协作改造的设计思路和后续维护要点,供继续对接其他 AI 工具(Codex / GPT 等)时参考。

## 1. 技术栈与运行

- 基于 Fuwari 模板修改:Astro 5 + Svelte 5 + Tailwind CSS 3 + Stylus + Pagefind + PhotoSwipe
- 包管理器 pnpm:`pnpm install` → `pnpm dev`(开发)/ `pnpm build`(构建,含 Pagefind 索引)/ `pnpm preview`(生产预览)
- 搜索是**纯静态实现**:Pagefind 在构建时生成索引(`pnpm build` 自动执行),前端运行时加载,不需要任何服务器。索引文件在 `dist/pagefind/`。
- 内容完全数据驱动:新文章只需在 `src/content/posts/` 加 Markdown;分类、标签、归档、RSS、侧栏计数全部自动生成。

## 2. 核心设计语言:新拟物(Neumorphism 2.0)

### 2.1 统一光源与凹凸语义(最重要,不要破坏)

所有组件共享同一光源:**左上高光、右下阴影**。凹凸不是装饰,是语义:

| 元素 | 凹凸 | 例子 |
| --- | --- | --- |
| 容器外壳 | 稳定凸起(外阴影) | 卡片、导航栏、侧栏 |
| 输入/屏幕/读数 | 下沉(内阴影) | 搜索框、统计槽、字数读数、图片相框内槽 |
| 可点击控件 | 凸起,悬停抬升,按下转内阴影 | 按钮、分类行、标签、分页、前后文章导航 |
| 当前选中 | 下沉或彩色实体指示 | 分页当前页 |

改任何组件前先想清楚它是"外壳、槽、还是键",再套对应的阴影组合。

### 2.2 设计令牌(design tokens)

统一在 `src/styles/variables.styl` 的 `define({})` 中管理(浅色/深色双值):

- 表面层级:`--page-bg`(页面背景)→ `--card-bg`(外壳)→ `--surface-high`(高位/更亮)→ `--surface-low`(下沉槽/更暗)
- 光影:`--neu-shadow` / `--neu-shadow-strong`(右下)、`--neu-highlight` / `--neu-highlight-strong`(左上)、`--neu-inset-shadow` / `--neu-inset-highlight`(内槽)、`--neu-border`(边缘反光描边)
- 强调色:`--primary`(主色,跟随 `--hue`)、`--accent-cyan` / `--accent-coral` / `--accent-mint` / `--accent-amber`

**不要在组件里硬编颜色或阴影值**,一律引用令牌。常用复合样式已封装在 `src/styles/main.css` 的 `@layer components` 里:

- `.card-base` 外壳卡片 / `.btn-card` 大按钮 / `.btn-regular` 常规按钮
- `.neu-row` 分类行托盘 / `.neu-badge` 嵌入式数字窗 / `.neu-tag` 标签键帽 / `.neu-mini-tag` 文章元信息小标签
- `.neu-inline-link` 拟物行内链接(悬停凸起、按下下沉)
- `.post-facts` 下沉读数窗 / `.post-desc` 描述彩色侧线 / `.post-nav-*` 前后文章方向键
- `#post-cover`、`.post-cover-frame`、`markdown.css` 的 `img` 下沉相框

### 2.3 材质规则

- 卡片**不透明**,不许透出背后的头图/人物;毛玻璃只允许做背景氛围,不能当外壳。
- 深色模式用深灰蓝实体材质(不是黑色半透明),左上高光更弱但必须可见,不用廉价霓虹泛光。
- 背景光斑(Mica 式 radial-gradient)已调到很淡(2–5%),如果觉得"挡图片"优先继续降低这里,而不是删掉。
- **所有"发光"效果已移除**:LED 灯、指示灯、按钮都只有内高光和投影,没有外发光光晕(塑料件只反光、不发光)。新增指示灯时沿用 `inset 1px 1px 1px rgba(255,255,255,.4)` 的内高光写法,不要加 `0 0 0 Npx` 的 halo 环。

### 2.4 强调色的使用

- 主色默认 `--hue: 305`(蓝紫),用户可在导航栏调色按钮切换。
- 调色按钮:**单击循环预设色相**(35/78/165/220/250/305),桌面端**悬停**展开彩虹滑杆精调(`Navbar.astro` 脚本 + `DisplaySettings.svelte`,两者通过 `window` 的 `hue-cycle` 自定义事件同步)。
- 明暗切换:默认跟随系统(AUTO),**单击直接在亮/暗间切换**(`LightDarkSwitch.svelte`,AUTO 时按系统当前偏好取反);三态菜单仍保留在悬停面板里。
- 分类行 LED 灯和文章卡左侧灯条按**名称散列**自动取色(primary/cyan/coral/mint 四色循环),逻辑在 `Categories.astro` 和 `PostCard.astro`,新增分类自动配色,不要硬编固定板块的配色。
- 联系方式图标用品牌色(`Profile.astro` 的 `linkColors` 映射:B站蓝/TG蓝/邮箱珊瑚/GitHub 跟随文本色)。

## 3. 踩过的坑(后续维护务必注意)

1. **Tailwind/PostCSS 不允许 `@media` 里套 `@layer`**。一旦写了,媒体查询会被静默丢弃、规则全端生效(曾导致桌面端前后文章导航变单列、卡片灯条消失)。正确顺序:`@layer components { @media (...) { ... } }`(见 `main.css` 底部)。
2. **Fuwari 主网格的行高由左栏决定**(`lg:row-span-3` 技巧),主列内容实际溢出网格盒,不要把任何新元素放在 `#main-grid` 之后指望它跟在主列内容后面——页脚是放在 `#content-wrapper` 内 + `margin-top: auto`,用 `min-height: calc(100vh - var(--main-panel-top) - 1rem)` 把短页面的页脚推到视口底部(`MainGridLayout.astro` 的 `.content-column`)。
3. **Swup 换页会替换 DOM**,组件内 `addEventListener` 会失效。需要跨页面交互(如 CC 链接点击复制)时用 document 级事件委托 + 全局去重标记(见 `License.astro` 的 `__licenseCopyBound`)。
4. 移动端顶栏按钮组用 `gap-1.5 md:gap-2` 保持间距;文章卡在移动端也是独立卡片+间隙(`PostPage.astro` 的 `gap-3`),不要再合并成长条。
5. 标签/按钮的 `:active` 下沉阴影在深色模式下要收敛(2–3px),否则整颗按钮会变黑块;`:focus-visible` 统一用主色半透明描边,`:focus` 去掉默认 outline。
6. `prefers-reduced-motion` 已有兜底,新增动效保持 180–240ms、`cubic-bezier(0.22, 1, 0.36, 1)`、抬升 1–4px。
